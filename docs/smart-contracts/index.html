Smart Contracts (V3)

BlockchainTokenSniper V3 interacts with a combination of its own custom smart contracts and standard decentralized exchange (DEX) contracts to perform its functions. All user interactions for tier management and bot operations are handled via the @btsv3bot Telegram bot, which then communicates with these contracts on the backend.

BTSManagerV2 (Management & Tiers)

The BTSManagerV2 contract continues to play a role in V3, primarily for managing user tiers, processing tier payments, and handling aspects of the referral system. Users interact with these functions indirectly through the Telegram bot's menus.

  • Example (BSC Mainnet): 0x83804A48E21a499AebB4063c07cfAAA5cB37553f (Address may vary or be updated; always refer to official announcements if direct interaction were ever needed, though V3 is designed for bot-only interaction for users).

Key functions managed by or through this contract (via the bot):

  • Tier purchasing and upgrades.
  • Referral registration and commission payouts for tier sales.
  • Management of user wallet addresses associated with tiers.

TradingCheckerV1 (Pre-Trade Safety Checks)

BTS V3 utilizes a custom TradingCheckerV1 smart contract to perform pre-trade safety checks. This helps protect users by analyzing tokens for common issues before a buy transaction is attempted.

Source code for this contract can be found in the project's contracts/ directory (TradingCheckerV1.sol). Its ABI is located in ABI/TradingCheckerABI.json.

Key checks performed by this contract (invoked by the bot):

  • Honeypot detection (checks if a token can be sold after buying).
  • Buy/Sell tax analysis (ensures taxes are within user-defined limits).

DEX and Token Contracts (Standard Interactions)

For actual buying and selling of tokens, BTS V3 interacts directly with standard Decentralized Exchange (DEX) router contracts (e.g., UniswapV2-like routers such as PancakeSwap Router) and standard ERC20 token contracts.

The bot uses standard ABIs for these interactions, including:

  • ABI/ERC20ABI.json: For interacting with ERC20 tokens (balance checks, approvals).
  • ABI/UniswapV2ABI.json: For interacting with UniswapV2-compatible DEX routers (swapping tokens, adding/removing liquidity if ever needed by advanced features).
  • ABI/UniswapV2FactoryABI.json: For fetching pair addresses from DEX factories.
  • ABI/pairABI.json: For interacting directly with liquidity pair contracts (e.g., to get reserves).

The specific router addresses for supported DEXs on each chain are configured within the bot's network_config.json file and selected by the user (or automatically) through the Telegram interface.

In V3, the previous custom BTSRouterV2 contracts for direct trading have been largely superseded by direct interaction with standard DEX protocols, offering greater flexibility and compatibility.
